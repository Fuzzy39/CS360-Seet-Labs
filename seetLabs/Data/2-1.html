Even though this is fine when there's only one line to be run inside the conditional, it's almost always best to use curly braces like this:

We can extend our decision-making with else if and else clauses to handle multiple scenarios. Look at this program, and see if you can guess its output:

int my_race_time = 59;
int record_race_time = 51;
System.out.println("Your time was: " + my_race_time + " seconds");
if (my_race_time < 55) {
	System.out.println("Wow! That was fast!");
}
else if (my_race_time < 60) {
	System.out.println("Excellent!");
}
else if (my_race_time < 70) {
	System.out.println("Good!");
}
else {
	System.out.println("You finished the race!");
}
if (my_race_time < record_race_time) {
	System.out.println("A new record!!");
}
else {
	System.out.println("You were " + (record_race_time - my_race_time) + " seconds slower than the record.");
}

In this example, the output will be:
Your time was: 59 seconds
Excellent!
You were 8 seconds slower than the record

Looping

Sometimes, we need to repeat actions multiple times. Loops help us avoid writing the same code over and over. This makes code more reusable and expands the type of things we're able to do with our programs.

While Loops: One more time!

The while loop repeats as long as a condition is true: 

int count = 0;
while (count < 5) { 
	System.out.println(count); 
	count++; 
}

In this case, the body of the loop will be executed 5 times, once each when count is equal to 0, 1, 2, 3, and 4. When count becomes 5, the loop breaks since count is not less than 5. Here's another example of a while loop. Try and guess what this code is doing, and also what the output will be:

int value = 0;
while (value * value <= 100) {
	value++;
}
System.out.println(value);

This code searches for some value which has a square greater than 100. The first integer which fulfils this condition is 11.

For Loops: A range of possibilities!

The for loop is perfect when you know exactly how many times you want the code to be run. Here's an example of looping on a range: 

for (int i = 0; i <= 5; i++) { 
	System.out.println(i); 
}

This code will print all the integers between 0 and 5. The for loop is defined by three statements, separated by semicolons (';').
The first is the Initialization, where we can create variables to be used inside the loop. In the loop above, int i = 0 creates a variable called i with an initial value of 0. Most commonly, these are single-letter variables starting alphabetically with i, then j, and so on if needed (some programmers choose to use 'ii' instead of 'j'. Always make your code as understandable as possible!). Remember when we talked about variable scope? These loop variables are scoped only to the inside of the loop: they're destroyed as soon as the loop breaks.
The second statement is the Condition, which tells the code when to continue and when to exit or break. This looks the very same as the conditional inside an if or while block. Once the condition becomes false, the loop exits. In the example above, the condition is i <= 5. Once this condition is false (when i is 6 or above), the loop will stop executing. Note that the condition will only be evaluated after a loop is complete.
Finally, there is the Increment. This tells the loop what should change after each iteration. In the above example, i++ means that I should be incremented, or 1 added to it, after each loop execution. Like the condition, this will only be evaluated after the whole loop has been run.

Do-While Loops: Guaranteed Fun!

What if we want to use a loop, but we want to ensure the body is executed at least once? The do-while loop runs the code before it checks conditions, that way at least one execution is performed: 

int i = 0;
do { 
	System.out.println(i); 
	i++; 
} while (i < 5);

In this example, the loop will run once for i = 0, 1, 2, 3, 4 and 5. The loop will only check if i is greater than 5 after the body is executed. Here's an example where the benefits of a do-while loop are used: 

int numPeople = 1;
do {
	System.out.println("Welcome!");
	numPeople--;
} while (numPeople > 0);

In this example, even though the condition is false once we reach the end of the loop, the code was run one time. Do-While loops are like someone who insists on trying every flavor before choosing a favorite!

Understanding Loops

The for loop and the while loop are closely related: they could be siblings! Every for loop can be rewritted as a while loop:

for (int i = 10; i > 0; i--) { ... }

Is really the same as:

int i = 10;
while (i > 0) {
	...
	i--;
}

So, why do we use for loops? Well, for loops have a much more concise syntax: almost every programmer would prefer to read a for loop than a while loop, even when they theoretically do the same thing. Each has their own use, but clarity and readability are key in everything. Learn to use these loops well, and they'll be excellent tools.

Advanced Looping (For-Each)

Sometimes, we just want to loop through every item in a collection without the hassle of managing indexes. The for-each loop makes this easy: 

int[] numbers = {1, 2, 3, 4, 5};
for (int num : numbers) { 
	System.out.println(num); 
}

Each entry in the array will be printed in this example. It's quick, clean, and gets you on with the fun stuff. We'll return to for-in loops when we talk about Collections in a later module.

Break and Continue: The Loop Control Squad

Want to exit out of a loop early? Use break to exit the loop like you'd like to escape a boring meeting: 

for (int i = 0; i < 10; i++) { 
	if (i == 5) { 
		break; 
	} 
	System.out.println(i); 
}

Even though the condition is true when i equals 5, the inner break will be encountered and the loop will stop before printing 5. Here's another case: imagine you want to skip a part of the loop and move on, rather than completing all the rest of the body. Use continue for this kind of functionality: 

for (int i = 0; i < 10; i++) { 
	if (i % 2 == 0) { 
		continue; 
	} 
	System.out.println(i); 
}

In this code, 2 will not be print out, but 3, 4, and so on will be. It's like skipping past the boring part of a movie!

Nested Loops and Conditionals

We can nest loops and conditionals inside each other to tackle complex problems. For example: 

for (int i = 0; i < 3; i++) { 
	for (int j = 0; j < 3; j++) { 
		System.out.println("i: " + i + " j: " + j); 
	} 
}

This is when you might want to use 'j' as an index like we talked about before. In this code, the inner loop will be fully executed for each time the outer loop is run. A total of 9 lines will be print by this code.
Beware! nesting too many loops can make your code as tangled as a bowl of spaghetti! And nested conditionals? They're like decision trees in a magical forest- it's easy to lose the path: 

if (score >= 90) { 
	if (bonus) { 
		System.out.println("A+"); 
	} else { 
		System.out.println("A"); 
	} 
}

There are certainly uses for nested conditionals, but you should be careful to make your code clear and readable.
