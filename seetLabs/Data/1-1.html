Reading 2 : Variables

Variables

If you've paid attention in your math classes, you've heard of variables. Variables are essentially names for some number, whether you know its value or not. In Java, variables are a little more complicated, but the basic idea is the same. Variables store some bit of data so we can use it throughout our program. The most important things to know about a variable are its Name, Type, and Value. Here's how we create a variable in Java: 

int myVar = 10;

There are a few things to notice here. The first keyword we use is int, which tells Java what kind of variable we are making, in this case an integer. Next, we give the variable a name, myVar in this case, which can be anything we want so long as it starts with a letter ('A'-'Z' or 'a'-'z') or underscore ('_'). Finally, we give the variable a value. You'll notice that we use a regular, old equals sign = for that, which is called the Assignment Operator (how fancy). We could have skipped this part for now, but we have to make sure to give our variable a value at some point before we actually use the it for anything. We could have done the same thing this way: 

int myVar;
// Some other code
myVar = 10;

The first line here is called Declaration, when we tell the compiler to create a place to store a value, while the last line is called Instantiation, when we tell our program to give the variable a value. Next, we'll talk some more about Data Types.

Primitive Types

Every variable in a Java program has a Type. This tells the Compiler how to treat a variable, how big it is, and what kind of stuff we can store in it. Some of these types are called Primitive Types, which doesn't mean they're any less useful than other types but does mean that they hold a pure, simple value of some kind. In Java, there are 8 primitive types:
boolean — Hold true or false and nothing in between. boolean likesPizza = true;
byte — Holds 8 ones or zeroes (bits) for a range of [-128, 127]. byte pizzasEaten = 100;
char — Holds a single character, like the keys on your keyboard (as well as a bunch of other interesting symbols). To mark a char, use single quotes (' ') around one letter or symbol. char lastNameInitial = 'L';
short — A 16-bit whole number with a range of [-32768, 32767]. short totalPizzas = 32000;
int — A very commonly-used datatype holding a whole number with a range in the billions. int MAX_PIZZAS = 1000000;
long — Holds a whole number value with double the size of an int and a range in the quintillions. long population2020 = 7_855_000_000L;
float — Holds a decimal value with 7 digits of precision. float pepperoniPercentage = 0.40f;
double — Another very common type which holds decimals with 15 digits of precision. double pizzaWeight = 2.5934;
All of these data types have their own use and applications, but boolean, int, and double are the most commonly used primitive types.

Reference Types

As opposed to Primitive Types, Reference Types are not values themselves but rather point to a place in memory and tells the program where to find some other values, like how the number on your house tells people where to find it. Reference Types refer to Objects of which there are many, many varieties. Some of the most common are:
String — Holds a string of characters, like a sentence or a name. To create a string, use double quotation marks (" "). String pizzaType = "Cheese";
Date — This is used to store a specific point in time, given as a number of milliseconds since the beginning of the year 1970. Date birthday = new Date();
List, Set, and Map — These are Collections, which will be explored in a later chapter.
Integer, Double, Boolean — These are different from the primitive types we already talked about: notice the capital letters which mark these as Wrapper Classes. We'll explain these more later.
All of these types start with a capital letter and are made up of the primitive types we already discussed. They can also have certain Methods which define their behavior and interactions. For instance: 

String name = "Jimmy";
System.out.print(name.charAt(0)); // Prints 'J'

In this example, we create a String and then ask for the first character inside that string. Here's an example of another type of Reference Type:

Date myDate = new Date(0);
System.out.print(myDate.getTime()); // Prints "0"

In this case, we create a Date Object using the new keyword, and give it a value of 0. Then, we print that same value using the .getTime() method. All Objects have a set of defined variables (also called fields, attributes, or members), as well as a set of operations (also called methods, functions, or behaviors). One very important reference value is null, which is a reserved keyword representing the absence of a value or reference. We can set any reference value to null, but attempting to use that reference will result in an error:

Date date = null;
System.out.print(date.toString()); // ERROR!

Type Conversion and Casting

Sometimes, you might need to change the type of a variable, or to use its value as a different type. When using Objects which provide an easy way to do this, the process is simple. For instance, we might want to turn a String into an int:nt myInt = Integer.parseInt("100"); This will instantiate myInt with a value of 100. However, how can we turn a int into a double, and vice versa? This is called Casting, and can be performed in several ways. One type of type conversion is implicit casting, in which the compiler will automatically widen certain values into values with a greater range, size, or precision: 

int intValue = 10;
double doubleValue = intValue; // doubleValue is 10.0

In this case, the int is widened into a double. This works for float -> double, int -> long, short -> float, etc. Another type of type conversion is called explicit casting and must be used when performing narrowing, or turning a value into a value with a lesser size, range, or precision:

double doubleValue = 10.99;
int intValue = (int) doubleValue; // intValue is 10

The decimal value of the double will be chopped or truncated and lost, not rounded.

Constants

There are many reasons you might want to create a Constant Variable, the value of which will never change. For instance, many important mathematical values are constants, like π, e, or φ. In Java, we can make a constant using the final keyword:

final double PI = 3.141592653589793;

Once set, the value of a final variable cannot be changed, which means that you must initialize the variable on the same line as it is declared. Constants are accessed just like any other variable inside your programs. It is convention to name final variables with capital letters, with words separated by underscores.

Other Attributes of a Variable

While we've already discussed the most important attributes of a variable, there are many more ways to describe a variable than just its type, name, and value. These include: Scope, or the area of code in which a variable can be accessed; Mutability, or whether a variable's value can be changed (remember constants); Visibility, which describes what classes may access a variable (like public, private, and protected); Location, which we discussed with Reference Variables; and Lifespan, which is closely linked to scope and describes the period during which a variable exists (like local and static). All of these describe when, how, and where a variable may be set, modified, and accessed.
