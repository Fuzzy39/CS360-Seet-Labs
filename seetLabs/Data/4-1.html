Reading 4: Objects and Classes

Objects and Classes

Defining Classes

A class is a blueprint that defines the state (fields) and behavior (methods) of objects. In Java, declare a class with:

public class Car {

	// fields and methods

}

Classes group related functionality into a single unit.

Instance Variables and Methods

Instance variables (fields) store object data, and methods define actions. Example fields:

public class Car {

	String model; // e.g., "Tesla"

	int year; // e.g., 2025

}

Example method:

public void drive() {

	System.out.println("The car is driving.");

}

Constructors

A constructor initializes new objects. It has no return type and matches the class name. Example:

public Car(String model, int year) {

	this.model = model; // set instance field

	this.year = year;

}

If no constructor is defined, Java provides a default no-argument constructor.

Static Members

static fields and methods belong to the class itself rather than any instance. Use static for shared data or utility methods. Example:

public static int carCount = 0; // shared among all Car objects

public static void showTotalCars() {

	System.out.println("Total cars: " + carCount);

}

Access static members via Car.showTotalCars().

The %kthis%/k Keyword

this refers to the current object. It clarifies references when parameter names match field names:

public void setModel(String model) {

	this.model = model; // assigns parameter to field

}

Without this, Java treats model as the parameter.

Encapsulation

Encapsulation hides internal details by using access modifiers (private, public, protected). Provide getters and setters to control field access:

private String model;

public String getModel() { return model; }

public void setModel(String model) { this.model = model; }

Nested and Inner Classes

Java allows classes within classes. Static nested classes act like top-level classes but scoped inside another. Inner classes (non-static) have access to outer instance members. Use them for logically grouping related classes.

Best Practices

- Keep classes focused: each should have a single responsibility.
- Use private fields and public getters/setters for encapsulation.
- Limit use of static: prefer instance methods unless shared behavior is needed.
- Comment constructors and complex methods for clarity.
- Follow naming conventions: classes in PascalCase, methods and fields in camelCase.
